SQL Tuning Advisor Practice

Use SQL Tuning Advisor when the problem is a specific SQL statement. If the bottleneck is locking, commit latency, or application behavior, advisor output may be secondary evidence rather than the fix.
TopicRecommendationWhy It Matters
Partition pruningIf PAYMENTS_HISTORY is range partitioned by txn_date, add a date predicate and verify PSTART/PSTOP in the execution plan.Avoid scanning old partitions for same-day AML work.
Index strategyConsider local indexes on amount, beneficiary_country, and txn_date only after testing selectivity and DML cost.Reduce full-scan pressure where predicate is selective.
Workload isolationRun regulatory extracts on Active Data Guard, a reporting PDB, or a scheduled batch window.Protect customer-facing OLTP response time.
