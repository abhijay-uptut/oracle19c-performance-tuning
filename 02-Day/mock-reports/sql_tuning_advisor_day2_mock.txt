SQL Text
~~~~~~~~
SELECT /* day2_incident_customer_statement */ COUNT(*)
FROM (
  SELECT transaction_id, txn_ts, amount, status, channel
  FROM day2_txn_big
  WHERE customer_id = :B1
  AND txn_ts >= ADD_MONTHS(SYSDATE,-18)
  ORDER BY txn_ts DESC
  FETCH FIRST 200 ROWS ONLY
)

Findings
~~~~~~~~

Finding 1: Index Finding
Impact: 91.4% estimated benefit

The optimizer estimates that creating the following index will significantly
improve this statement.

Recommendation:

CREATE INDEX ABHIJAY.IDX_DAY2_TXN_STMT_FIX
ON ABHIJAY.DAY2_TXN_BIG("CUSTOMER_ID","TXN_TS" DESC);

Rationale:
The query filters by CUSTOMER_ID and TXN_TS, then orders by TXN_TS descending.
The current index on BRANCH_ID, TXN_TS does not support this access pattern.
The recommended index allows Oracle to find one customer's recent transactions
without scanning a large part of DAY2_TXN_BIG.

Validation:
Do not create the index directly in production.
Create it as invisible in a test session, gather stats, enable invisible indexes
for that session, and compare before/after metrics.
